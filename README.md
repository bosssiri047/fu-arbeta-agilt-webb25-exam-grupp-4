# Gruppreflektioner

Svara på var och en av frågorna nedan som grupp. MINST 100 tecken per fråga.

## Frågor

### Inledning

#### Sammanfatta projektets syfte, samt era gemensamma mål

Det gemensamma målet som grupp och syfte var att. Lära sig att arbeta Agilt som grupp och medföra en struktur där gruppen kan arbeta till ett avsatt mål. 

#### Hur upplevde teamet projektet initialt?

Gruppen uppfattade arbetet som spännande moment att utföra, då man kunde få implementera sin kunskap till ett gemensamt målarbete. Det skulle bli givande att få arbeta och utföra och uppleva hur merge och github skulle fungera. 

### Planering och genomförande

#### Beskriv hur ni inom teamet planerade projektet initialt

Vi gick igenom alla user stories och diskuterade vad som var viktigast och hur svårt något skulle vara att lösa. Kollade igenom vilka figma skisser som fattas. Vi pratade om hur vi skulle dela upp scripten, hur vi skulle få bas html/css i starten utan att det krockar etc. 

#### Vilka steg har teamet tagit för att utveckla applikationen?

Följde scrum strukturen.
Vi började först med lofi skisser för sidorna som fattades. Sedan tog vi först uppgifter för att bygga upp det statiska och lätta som index, foodtruck nav meny etc så vi hade en bas att börja på. Sedan plockade vi uppgifter vi kände att vi kunde ta oss an. När någon blev klar med sin gick vi tillsammans och löste merge's. Sedan testar vi att det som vi kodade funkar tillsammans ex: meny sidan och kundvangen, ev gör någon patch om det är något som strular. Sedan väljer nästa uppgift.

### Teamets utmaningar och lösningar

#### Vika var de största utmaningarna för teamet?

Git merge, hur man läser av github och vs code hur man mergar. Tex. Accept incoming, accept both changes, current incoming. 
Get merge gå igenom det till summans, sedan eget. 
Vincent del av att be om hjälp, som inkom vid frågande i discord. 

#### Hur löste eller hanterade teamet dessa utmaningar?

Gick tillsammans som grupp för att lösa merges så att vi alla lär oss och kan hjälpa till. Vid påminnelse att det tillhör Agilt arbete i discord att hjälpa varandra så frågade Vincent om hjälp och vi löste det. 

#### Vilka kompromisser inom teamet har ni gjort under projektets gång?

Under första sprinten så väntade vi på att Vincent skulle göra klart kvitto sidan, han frågade om hjälp och vi löste det han var fast på men han hann inte göra klart den i tid så Boss gick in och gjorde en snabbt för våran mvp. Vi kompromissa lite med utseendet på appen för att vi ville få klart så mycket av backloggen som möjligt eftersom vi bara var 3 i teamet.

### Teamets reflektion och utvärdering

#### Vad lärde sig gruppen inom projektet?

Kommunikation är superviktigt så att alla i teamet ser hur projektet ligger till och att man är på samma bana. Att fråga om hjälp så att arbetet kan flöda på utan hinder. Skriva kommenterar i koden så att alla i teamet lättare och mer effektivt kan förstå och bygga på varandras kod. Vikten av att använda branches så att merges blir lättare att hantera.

#### Hur kan gruppen förbättra sitt samarbete framöver?

Ge mer tid till planeringsfasen och gå igenom kodstruktur, namngivning, bygga modulära funktioner, kommentera. Att inte vara rädd att be om hjälp och att kunna anpassa sig till situationer som dyker upp. Att inte vara för fast i sin egen lösning och att det är hela tiden produkten i sig som är i fokus.
